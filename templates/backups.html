On for MinIO and most self-hosted gateways (host/bucket/key); off for AWS (bucket.host/key). Getting it wrong shows up as a signature error, not a 404.

Credentials

Backup encryption key

This key is the only thing that can decrypt your backups. It is not stored in the hub database, and it is not recoverable — if this server is lost and you have not written this down somewhere else, every backup ever taken becomes unreadable.

Store it in a password manager or a sealed envelope — somewhere that does not depend on this machine. To restore, you need this key and the backup file, and nothing else:

python restore_backup.py --in <file>.fhb --out temp_v2.db --verify
